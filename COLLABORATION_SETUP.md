# Инструкция по настройке совместной работы над проектом

## Вариант 1: Git + GitHub (Рекомендуется) + Live Share

Этот вариант позволяет работать вместе, видеть изменения друг друга и иметь историю изменений.

### Шаг 1: Инициализация Git репозитория

1. Откройте терминал в папке проекта
2. Выполните команды:

```bash
# Инициализация Git репозитория
git init

# Добавление всех файлов
git add .

# Первый коммит
git commit -m "Initial commit"
```

### Шаг 2: Создание репозитория на GitHub

1. Зайдите на https://github.com
2. Создайте новый репозиторий (New Repository)
3. Назовите его, например: "new-games-project"
4. НЕ добавляйте README, .gitignore или лицензию (они уже есть)
5. Скопируйте URL репозитория (например: `https://github.com/ваш-username/new-games-project.git`)

### Шаг 3: Подключение к GitHub

```bash
# Добавьте удаленный репозиторий (замените URL на ваш)
git remote add origin https://github.com/ваш-username/new-games-project.git

# Отправьте код на GitHub
git branch -M main
git push -u origin main
```

### Шаг 4: Добавление друга как соавтора

1. На странице репозитория на GitHub
2. Перейдите в Settings → Collaborators
3. Добавьте GitHub username вашего друга
4. Друг получит приглашение и сможет клонировать проект

### Шаг 5: Друг клонирует проект

Ваш друг должен выполнить:

```bash
# Клонирование проекта
git clone https://github.com/ваш-username/new-games-project.git

# Переход в папку проекта
cd new-games-project

# Установка зависимостей
npm install
```

### Шаг 6: Работа с Git (ежедневная работа)

**Когда вы делаете изменения:**

```bash
# Посмотреть изменения
git status

# Добавить измененные файлы
git add .

# Создать коммит с описанием
git commit -m "Описание изменений"

# Отправить на GitHub
git push
```

**Когда друг делает изменения, вам нужно получить их:**

```bash
# Получить изменения с GitHub
git pull
```

**Важно:** Всегда делайте `git pull` перед началом работы, чтобы получить последние изменения!

## Вариант 2: Live Share в Cursor (Реальное время)

Этот вариант позволяет редактировать код одновременно в реальном времени.

### Настройка Live Share в Cursor:

1. **Установите расширение Live Share:**
   - Откройте Cursor
   - Перейдите в Extensions (Ctrl+Shift+X)
   - Найдите "Live Share" от Microsoft
   - Установите его

2. **Начните сессию совместной работы:**
   - Нажмите F1 или Ctrl+Shift+P
   - Введите "Live Share: Start Collaboration Session"
   - Скопируйте ссылку и отправьте другу

3. **Друг присоединяется:**
   - Откройте Cursor
   - Установите расширение Live Share
   - Нажмите F1 → "Live Share: Join Collaboration Session"
   - Вставьте ссылку

**Преимущества:**
- Видите курсор друг друга в реальном времени
- Можете редактировать одновременно
- Видите изменения мгновенно

**Недостатки:**
- Нужно быть онлайн одновременно
- Нет истории изменений (нужен Git)

## Рекомендуемый подход: Комбинация Git + Live Share

**Используйте оба варианта вместе:**

1. **Git + GitHub** - для:
   - Сохранения истории изменений
   - Работы в разное время
   - Резервного копирования
   - Отслеживания изменений

2. **Live Share** - для:
   - Совместной работы в реальном времени
   - Обсуждения кода
   - Быстрого решения проблем вместе

### Рабочий процесс:

1. **Утром:** `git pull` - получить последние изменения
2. **Во время работы:** Используйте Live Share для совместного редактирования
3. **После работы:** `git add .` → `git commit -m "..."` → `git push`

## Полезные команды Git

```bash
# Посмотреть статус
git status

# Посмотреть изменения
git diff

# Посмотреть историю
git log --oneline

# Создать новую ветку для эксперимента
git checkout -b feature-name

# Вернуться к основной ветке
git checkout main

# Объединить ветку
git merge feature-name
```

## Решение конфликтов

Если оба изменили один файл:

1. Git покажет конфликт при `git pull`
2. Откройте файл, найдите метки `<<<<<<<`, `=======`, `>>>>>>>`
3. Выберите нужный код или объедините оба варианта
4. Сохраните файл
5. Выполните: `git add .` → `git commit -m "Resolve conflict"` → `git push`

## Важные файлы для Git

Убедитесь, что `.gitignore` содержит:

```
node_modules/
dist/
*.log
.env.local
.DS_Store
```

Это предотвратит загрузку ненужных файлов на GitHub.
